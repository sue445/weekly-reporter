play環境構築---------最新版を落とすhttp://www.playframework.org/download---------http://d.hatena.ne.jp/daicham/20110921/1316600869を参考に~/Library/Javaに入れる---------githubのリポジトリhttps://github.com/sue445/weekly-reporter---------http://change-the-world.heroku.com/heroku周りのインストールをする---------ローカルにcloneするgit clone git@github.com:sue445/weekly-reporter.git---------Macのgithubクライアントでリポジトリ管理するとherokuにデプロイできなくなるっぽいのでやらないほうがいい（git push heroku masterしてもローカルのgitリポジトリにcommitされるだけ・・・）---------play ecでeclipse周りの設定を行う（http://www.playframework.org/documentation/1.0.1/ideに、eclipse関係の設定はチームで共有するなとあるのでgitignoreしてます）----------gitのremoteにherokuを追加git remote add heroku git@heroku.com:weekly-reporter.git----------gitにデプロイするgit remote heroku master----------webを見るhttp://weekly-reporter.herokuapp.com/----------[お約束]masterで作業をしない。必ず作業用にbranchを作ってそこで作業をする。git checkout -b my-working-branch自分の作業がテストまで完全に終わったらmasterに取り込む（masterはherokuに直接デプロイするものなので、不完全なものをmasterに取り込んだらみんなが迷惑します）git checkout mastergit merge my-working-branch または　git rebase my-working-branch
